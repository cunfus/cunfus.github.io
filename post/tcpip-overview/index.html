TCP/IP Overview

TCP/IP 实际上由一组协议构成，其中有两个最为重要，

  • IP 协议，在网络中，提供寻址（addressing）、数据报路由（datagram routing）和其他功能。
  • TCP 协议，在设备进程之间，提供连接建立（connection establishment）管理（management）和可靠的数据传输（reliable data transport）。 实际上，一个功能齐全的网络，需要各种不同的协议和技术共同作用。但鉴于它两的重要性，整个网络层和传输层的套件，被缩写成了 TCP/IP。

早期历史

1973 年，这部分还只有一个协议，TCP。经过数年的测试和开发，在 1977 年，TCP version2 出现了。这一年，Jon Postel，提出我们搞砸了，不应该用 TCP 既充当端到端协议，又充当网络打包路由协议，应该分层！ TCP 严格作为端到端协议划分到传输层，IP 划分到网络层。于是，TCP/IP 架构出现了。这个划分的过程，开始于 1978 年的 TCP version3 中。1980 年，现代网络中使用的 TCP 和 IP 的第一个正式的版本 v4 创建了，这就是为什么 IP 的第一个版本是 v4，而不是 v1。现在，它已经是全球的标准。

宏观图景

TCP/IP 架构涵盖了 OSI 模型中的许多层，从概念上可以将其分为两类：

  • 提供给其他 协议 的的服务 也就是核心协议，IP TCP UDP。IP 在网络层提供，寻址（addressing）、分发（delivery）、数据报打包(datagram packaging)、分段（fragmentation）、重组（reassembly）。在传输层，TCP 和 UDP 关心封装用户数据（encapsulate user data）、管理设备连接（manage connections）。

  • 提供给最终用户的服务 如应用层的 HTTP 协议，它为用户提供了互联网的 WWW 服务。FTP 协议，传输文件。SMTP 协议，发送邮件。

OSI 参考模型和 TCP/IP 参考模型1

TCP/IP 层次的划分，虽然相比 OSI 模型，损失了一些细节，但也是一件好事。本来高层功能的划分就相当随意，很难区分某些协议是五层、六层还是七层。

客户端/服务器

TCP/IP 的目的是为了数据交换，由此产生了 Client/Server 概念：

数据交换发起请求（request）的一方是客户端，响应（response）请求的一方是服务器。即使在 Peer-to-Peer 模型中，它的通信实际上仍然是 Client/Server，记住这一点很重要。

TCP/IP 协议

有时候，称 TCP/IP 为 protocol，而不是 protocol suite。虽然技术上不正确，却被广泛使用。实际上，它们是协议套件。
