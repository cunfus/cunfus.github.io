“学习” Python 的过程

没有什么“学习”，只有玩得开心。

周末两天，我简单学下了 Python。

大致情况呢，就是去官方文档和一些备受好评的教程里，整里了下要了解的东西，然后按照 对待拥有的时间 里方法，一个小阶段、一个小阶段去拿成果，并归纳在 笔记 里。

我接受的教育里，“学习”要正襟危坐，分析细节，理解细节（是的，不去质疑，全盘接受）。编程不是这样，更不要说是 Python 这样灵活的动态语言了。学习要去交流，去跟它对话，运行一下它，看看结果，思考思考。

当较为全面了解了 Python 后，我发现以前的思路过于狭窄，好像一直在抠一些没用的细节。感叹下，如果刚接触编程时，我第一个碰见的是 Python 就好了。这不是夸大其词，思维层面上的自由度、转化到代码的速度、接受的反馈，三者叠加带来的强烈的增益，更别提丰富的第三方库和跨语言接口了。

以前眼界太窄，才会执着什么都用 C 死磕出优美的代码，这也跟工作经历有关吧。

等我用 Python 做一段时间项目，再继续接触接触其他语言吧，有了几个备选了。

  • Prolog 推断逻辑关系的语言
  • Lisp 更强大的抽象能力
  • …
