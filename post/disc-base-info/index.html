Disc-基本词汇扫盲

宽高比

4:3 是早期电视和电影的标准宽高比，最初的电视机和电影胶卷采用这种比例。

随着技术的发展，尤其是高清电视的出现，16:9 成为新标准。该比例更适合现代电影和宽屏显示，符合人眼的视觉体验。

分辨率

  • SD (Standard Definition) 480p，720 x 480，标准清晰度，宽高比 4:3 或 16:9
  • qHD（Quarter HD），540p，1280 x 540，较低的高清晰度格式
  • HD (High Definition) 720p，1280 x 720，高清分辨率，宽高比 16:9
  • Full HD (FHD) 1080p，1920 x 1080，全高清，宽高比 16:9
  • 2K UHD（Ultra High Definition），2048 x 1080，宽幅，宽高比 17:9
  • QHD (Quad HD) 1440p，2560 x 1440，四倍于720p的清晰度，宽高比 16:9
  • 4K UHD，2160p，3840 x 2160，超高清，宽高比 16:9。
  • 8K UHD 4320p，7680 x 4320，超高分辨率，宽高比 16:9。
  • 其他分辨率
    • VGA（Video Graphics Array），640 x 480，宽高比 4:3
    • WVGA（Wide VGA），800 x 480，宽高比 15:9
    • other，1080 x 458，宽高比 2.35:1

从视频中截取一个画面，这个画面的分辨率是多少多少。

帧率

帧率（Frame Rate），也称为 帧频，是指视频中每秒显示的帧数，通常以 fps（frames per second）作为单位。常见的帧率有 24fps（电影）、30fps（电视节目） 或 60fps（游戏）。

从视频中截取一段场景，这个场景有多少副画面。

编码格式

编码格式（或称 编码标准）是指用于压缩和存储音视频数据的技术和方法。不同的编码格式适用于不同的应用场景，影响着视频质量、文件大小和播放兼容性。

常见的编码格式，

  • MPEG-4，早期的压缩标准，仍被广泛使用。兼容性好，但相对于新标准，压缩效率较低。
  • H.264 (AVC，Advanced Video Coding)，高压缩率和良好的视频质量。兼容性强，适用于大多数设备。
  • H.265 (HEVC，High Efficiency Video Coding)，比 H.264 提供更高的压缩效率，能够在相同质量下减小文件大小。更高的压缩率，但对硬件要求较高。
  • VP9（Video Processing version 9），Google 开发的开放编码格式，专为网络视频优化。压缩效率良好，支持高分辨率视频。
  • AV1（AOMedia Video 1，Alliance for Open Media），新一代开放编码格式，旨在替代 VP9 和 HEVC，尤其在高分辨率和低带宽条件下表现优越。压缩率高，但编码和解码的计算需求较高。
  • ProRes，苹果公司开发的编码格式，常用于制作高质量视频，但文件大小较大。

比特率

比特率（Bitrate）是指在单位时间内传输或处理的数据量，通常以 比特每秒（bps，bits per second）为单位。

  • 恒定比特率（CBR，Constant Bitrate）在整个视频中保持相同的比特率。适合于需要稳定带宽的应用，如直播。
  • 可变比特率（VBR，Variable Bitrate）根据视频的复杂程度动态调整比特率。复杂场景使用更高的比特率，简单场景则降低比特率。这种方式能在保持较高质量的同时，减少文件大小。

这一项并不是单独的指标，一个视频的比特率类型由分辨率、帧率、编码格式决定。CBR 提供了稳定性和简单性，适合实时应用；而 VBR 提供了灵活性和更好的画质，适合于视频制作和存储。

再由编码格式支持 CBR 或 VBR。

视频容器格式

这里是最熟悉的视频格式（多媒体容器格式）了。

  • MP4 (MPEG-4 Part 14)，支持多种编码格式，如 H.264、H.265（视频）和 AAC、MP3（音频）。有损（Lossy）。
  • MKV (Matroska Video)，开放源代码的容器格式，支持多种视频和音频编码。可以包含多个音轨、字幕轨道和章节信息。具体取决于所使用的编码格式（如 H.264 为有损，FFV1 为无损）。
  • AVI (Audio Video Interleave)，由微软开发，支持多种编码格式，但通常文件较大，兼容性较差。具体取决于所使用的编码格式（如 DivX 为有损，Uncompressed AVI 为无损）。
  • WMV (Windows Media Video)，由微软开发，专为流媒体和下载而设计，通常用于 Windows 环境。有损（Lossy）
  • MOV，由苹果公司开发，常用于 QuickTime 媒体播放器，支持多种编码格式，适合专业视频编辑。具体取决于所使用的编码格式（如 H.264 为有损，Apple ProRes 为无损）。
  • FLV (Flash Video)，主要用于 Adobe Flash Player，曾广泛用于在线视频，但现在逐渐被 MP4 等格式取代。有损（Lossy）。
  • WEBM，专为网络视频设计，通常用于 HTML5 视频播放，支持 VP8、VP9 编码。有损（Lossy）
  • OGG，开放格式，通常用于音频，但也可以包含视频，支持多种编码格式。Ogg Vorbis 为有损，Ogg FLAC 为无损。

还有些音频格式，等以后遇见了再补充吧。
