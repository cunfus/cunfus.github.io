70 - 爬楼梯

动态规划

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

解题思路：

假设有 k 个台阶，由于你有两种到达方式，爬一步 和 爬两步，得出：

f(k) = f(k - 1) + f(k - 2)，f(1) = f(0) = 1

递归

class Solution:
    def climbStairs(self, n: int) -> int:
        if n < 2:
            return 1
        else:
            return self.climbStairs(n - 1) + self.climbStairs(n - 2)

计算耗时呈指数级上升，得优化，存储中间结果。

class Solution:
    def climbStairs(self, n: int) -> int:
        def dfs(i, store):
            if i < 2:
                return 1
            if store[i] == 0:
                store[i] = dfs(i - 1, store) + dfs(i - 2, store)
            return store[i]
        return dfs(n, [0] * (n + 1))

递归不是 Python 的强项，n 超过 1000 这个数量级，会遇到栈溢出的问题，maximum recursion depth exceeded，递归限制。 因此，得用动态规划 DP。

class Solution:
    def climbStairs(self, n: int) -> int:
        dp = [0] * (n + 2)
        dp[0] = dp[1] = 1
        
        for i in range(2, n + 1):
            dp[i] = dp[i - 1] + dp[i - 2]
        
        return dp[n]

用了一个数组存储，空间复杂度 O(n)，其实不需要这么多。可以继续优化。

class Solution:
    def climbStairs(self, n: int) -> int:
        prev = curr = 1
        for i in range(2, n + 1):
            prev, curr = curr, prev + curr 
        return curr

但由于 Python 的执行环境是名称绑定，实际上这个空间消耗并不比直接开辟等量数组空间少。在 C/C++ 中这个优化是有效的。
