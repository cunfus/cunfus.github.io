Git

不常用的 git 操作

其他

  • git reset HEAD <file> 文件从 Stage 移除
  • git blame 查看代码是谁写的
  • git show 查看 commits tags 内容

git tag

  • git tag <version> 打标签
  • git push origin <version> 推送标签
  • git tag -d <version> 删除本地标签
  • git push origin :refs/tags/<version> 删除远程标签

git add -p

git add -p file 进入交互式 Stage 模式

例如： (1/3) Stage this hunk [y,n,q,a,d,j,J,g,/,e,?]?
y 表示暂存当前的修改
n 表示不暂存当前的修改
q 表示退出暂存模式
a 表示暂存当前以及其后面的所有修改
d 表示不暂存当前以及其后面的所有修改
j 和 J 表示移动到下一个修改
k 和 K 表示移动到上一个修改
g 表示跳转到指定行号
/ 表示搜索匹配的内容
e 表示手动编辑暂存的内容
? 表示显示帮助信息

git rebase -i HEAD~num

git rebase -i HEAD~2 进入交互式 rebase 模式，重新整理 Commit history

1. 列出一个 commit 列表，每个 commit 前面有一个操作指令，默认是 pick
2. 按你的想法修改操作指令，保存并关闭文件
3. 紧接着打开一个新的界面，用于编辑合并后的 commit 信息，修改保存并关闭
4. 如果你要覆盖远程仓库 `git push origin branch-name --force`

另一种整理 Commit history 的思路如下（符合重构之道）：

git checkout --orphan new-branch     # 创建一个新的孤立分支
git add -A                           # 将所有文件添加到暂存区
git commit -m "Your commit message"  # 提交变更
git branch -D master                 # 删除原来的分支（谨慎操作，确保没有重要的内容）
git branch -m master                 # 将当前分支重命名为 master（如果需要）
git push origin master --force       # 强制推送到远程仓库
